# Generic terminology conventions

> **Scope of this table:** Generic tech-writing conventions only — spelling,
> casing, and synonym choices that apply regardless of product or vendor.
> Product and feature names are **not** listed here; they come from the wiki's
> `terminology/` pages and the optional project term-table override in the
> Documentation Domain Context (CLAUDE.md / AGENTS.md).

Apply these conventions alongside the style rules; cite the relevant `R-…` ID
(usually `R-VOICE-05`, `R-GRAM-08`, or `R-INCL-02`) when flagging a violation.

| Preferred | Avoid | Note |
|---|---|---|
| Wi-Fi | WiFi, wifi | Trademark-correct spelling |
| internet | Internet | Lowercase; follow AP and MS style |
| email | e-mail | Closed compound; no hyphen |
| login (noun / adjective) | login (as verb) | "Use the **login** page" |
| log in (verb) | login (as verb) | "**Log in** to continue" — two words for the action |
| GHz, MB, GB, TB | Ghz, mb, gb, tb | SI prefix + unit casing matters; see R-NUM-07 |
| LAN, USB, PDF, URL | lan, usb, pdf, url | Always all-caps (established acronyms) |
| .NET 3.5, .NET 4.x | dotnet, Dot Net | Trademark-correct casing |
| TLS 1.2 | TLS1.2, TLSv1.2 | Standard's casing: acronym + space + version |
| Windows 10, Windows 11 | Win10, Win11 (body text) | Short forms acceptable in tables only |
| Microsoft Windows | MS Windows | Use in full at first mention in a document |
| select | choose | Prefer "select" for menu and list choices; see R-PROC-11 |
| click | press, hit | For pointing-device actions on screen elements |
| enter | type | "Enter" implies committing with the Enter key; see R-PROC-11 |
| make sure that | ensure | Prefer in procedural steps for plain-language clarity |
| recommended | suggested | When guidance originates from the product team |
| activate | enable | Use "activate" for licenses; "enable" for OS features and settings |
| allowlist | whitelist | Inclusive form; see R-INCL-02 |
| blocklist | blacklist | Inclusive form; see R-INCL-02 |
| primary | master (for device or server roles) | Inclusive form; see R-INCL-02 |
| secondary | slave (for device or server roles) | Inclusive form; see R-INCL-02 |
